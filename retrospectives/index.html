Retrospectives

Throughout a program it is important to inspect and adapt in order continuously improve our program execution. Program retrospectives are scheduled during the program for this purpose.

Milestone Retrospectives

  • Each program will have a retrospective board for a milestone​
  • Anyone can add to the milestone retro board​
  • The EPM will review the retro board on a sprintly basis looking for any red flags or issues that have come up​
  • The EPM will schedule end of milestone retrospectives​
  • All teams that took part in the milestone, as well as Product, should have a representative present​

End of Program Retrospectives

At the close of the program, a program wide retrospective is scheduled for all of the teams involved. All EPMs attend to see any issues that might be occurring in other programs.
